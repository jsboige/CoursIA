## Scope

The module-level singleton `get_verifier()` cached the LeanVerifier instance at the **first** `project_dir` seen. Any subsequent call with a different `project_dir` returned the same instance — **silent cross-project lock-in** (build files of the second project in the first project's Lake root). Surfaced by the depth-3 sweep in #6067 (See #1453). Fix: singleton → dict indexed by `project_dir`, plus `reset_verifier()` test escape hatch.

## Fix diff essence

```python
# MAIN (bug): singleton
_verifier = None
def get_verifier(project_dir=None):
    global _verifier
    if _verifier is None:
        _verifier = cls(project_dir=project_dir or LEAN_PROJECT_DIR, verbose=False)
    return _verifier  # ← always the FIRST instance, ignores later project_dir

# BRANCH (fix): dict per project_dir
_verifiers: Dict[str, "LeanVerifier"] = {}
def get_verifier(project_dir=None):
    global _verifiers
    pd = project_dir or LEAN_PROJECT_DIR
    cached = _verifiers.get(pd)
    if cached is not None:
        return cached
    instance = cls(project_dir=pd, verbose=False)
    _verifiers[pd] = instance
    return instance

def reset_verifier(project_dir=None):  # test escape hatch
    if project_dir is None: _verifiers.clear()
    else: _verifiers.pop(project_dir, None)
```
