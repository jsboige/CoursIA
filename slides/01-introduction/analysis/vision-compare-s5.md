## Critere 1 — Layout / Structure

| | A — Slidev | B — PPTX |
|---|---|---|
| Format | Plein ecran 16:9, marges larges | 4:3 compact, marges etroites |
| Organisation | Titre + filet + intro + 2 blocs bullets | Titre centré + numerotation slide + 4 blocs bullets |
| Profondeur | 2 niveaux (principal + sous-points) | 3 niveaux (principal + secondaire + tertiaire) |

**Vainqueur : A (Slidev)**
La structure deux-colonnes est lisible et l'espace est bien utilise. Le PPTX empile 4 blocs a 3 niveaux de profondeur dans un format 4:3 — la hierarchie devient difficile a lire. Slidev a reduit le contenu a 2 blocs et 2 niveaux, ce qui est plus clair.

## Critere 3 — Sous-titres / Hierarchie

| | A — Slidev | B — PPTX |
|---|---|---|
| Niveau 1 | Puces grasses bordeaux, tres visibles | Puces fleche, texte gras, mixte bold/normal |
| Niveau 2 | Puces cercle vides, taille normale | Puces carre/losange, taille reduite, italique |
| Niveau 3 | Absent | Puces tiret, encore plus petit, italique |
| Contraste hierarchique | Fort (gras bordeaux vs texte gris) | Moyen (3 niveaux sur format 4:3 = surcharge) |

**Vainqueur : A (Slidev)**
Slidev maintient 2 niveaux clairs avec un contraste visuel fort. Le PPTX tente de faire passer 3 niveaux dans un format reduit — le niveau 3 (Conditions reelles, Systeme complet, Comment emerge...) est quasiment illisible en projection.

| | A — Slidev | B — PPTX |

| Taille des polices | Grande (titre ~40pt, corps ~22pt) | Petite (titre ~24pt, corps ~14pt niveau 3) |

La typographie de Slidev est plus homogene et plus lisible. Le seul point negatif est la perte d'accents ("varies" au lieu de "variés", "strategiques", "Modelisation") — bug de rendu Slidev/theme a corriger. Le PPTX melange serif/sans-serif de facon desordonnee.

## Verdict final

**Vainqueur global : A (Slidev) — 7/7 criteres**

Slidev est superieur sur tous les criteres pour une projection en amphitheatre. Le PPTX souffre de trois defauts majeurs : fond gris qui tue le contraste, 3 niveaux de hierarchie illisibles en projection, et densite de contenu excessive en format 4:3. La conversion Slidev a fait des choix judicieux en reduisant le nombre de points et en adoptant un fond clair.

**Cependant** : Slidev a deux faiblesses a corriger — la grande zone vide en bas et les accents manquants.

## Top 3 corrections a appliquer a Slidev

### 1. Corriger les accents manquants (priorite haute)

Les puces de niveau 2 perdent les diacritiques francophones. Localiser dans `slides.md` et corriger :

```markdown
<!-- Actuel -->
- Recherche de solutions et jeux strategiques
- Modelisation probabiliste
- Concevoir des programmes intelligents dans des domaines varies :

<!-- Corrige -->
- Recherche de solutions et jeux stratégiques
- Modélisation probabiliste
- Concevoir des programmes intelligents dans des domaines variés :
```

Verifier aussi le fichier source et le theme pour tout encodage UTF-8 manquant.

### 2. Utiliser l'espace vide en bas (priorite moyenne)

La zone basse (~30% de la slide) est completement vide. Deux options :

**Option A — Reduire le padding bas du layout :**
```css
/* Dans le theme ou le layout de cette slide */
.slidev-layout.default {
  padding-bottom: 2rem; /* au lieu de auto */
}
```

**Option B — Ajouter un element visuel ancrant (icone ou tag thematiques) :**
```markdown
<div class="absolute bottom-8 right-8 text-4xl opacity-20">
  🧠
</div>
```
(Si les emojis sont autorises — sinon une barre de couleur ou un pictogramme SVG.)

### 3. Renforcer la lisibilite de la phrase d'introduction (priorite basse)

La phrase "A l'issue de ce cours, vous serez capables de :" est en texte courant gris, peu differenciee du reste. La rendre plus visuelle :

```markdown
<!-- Avant -->
A l'issue de ce cours, vous serez capables de :

<!-- Apres — avec style distinct -->
<p class="text-lg font-semibold text-gray-500 mb-4">
  A l'issue de ce cours, vous serez capables de :
</p>
```

Ou en CSS du theme :
```css
.intro-sentence {
  font-style: italic;
  color: var(--color-accent);
  font-size: 1.1em;
  margin-bottom: 1.5rem;
}
```
